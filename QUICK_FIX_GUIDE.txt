THE PROBLEM
═════════════════════════════════════════════════════════════════════════════

Your system shows:
  ⚠ "No AI providers available. Please configure at least one API key."

This means the system cannot find any API keys in your environment.

QUICK FIX (5 MINUTES)

STEP 1: Create .env File
────────────────────────────────────────────────────────────────────────────

Option A - Copy and edit existing config.env:
  1. Open: config.env (in your project folder)
  2. Find the [API KEYS] section
  3. Replace the placeholder values with YOUR ACTUAL API KEYS
  4. Save the file

Example:
  OPENAI_API_KEY=sk-proj-YOUR_REAL_KEY_HERE
  ANTHROPIC_API_KEY=sk-ant-YOUR_REAL_KEY_HERE
  DEEPSEEK_API_KEY=sk-YOUR_REAL_KEY_HERE
  GOOGLE_API_KEY=YOUR_REAL_KEY_HERE

Option B - Create new .env file in terminal:
  
  cat > .env << 'KEYS'
  OPENAI_API_KEY=sk-proj-YOUR_KEY_HERE
  ANTHROPIC_API_KEY=sk-ant-YOUR_KEY_HERE
  DEEPSEEK_API_KEY=sk-YOUR_KEY_HERE

TROUBLESHOOTING
═════════════════════════════════════════════════════════════════════════════

Q: Still getting the error?
A: Make sure:
   ✓ .env file exists in the same folder as your Python script
   ✓ load_dotenv() is called at the top of nova_ai_system.py
   ✓ API keys in .env are not wrapped in quotes
   ✓ No typos in environment variable names (must be UPPERCASE)

Q: Keys work individually but error persists?
A: Check the order of imports. load_dotenv() must be called BEFORE
   any other code that uses the keys.

Q: Getting "module 'python-dotenv' not found"?
A: Install it with:
   pip install python-dotenv
